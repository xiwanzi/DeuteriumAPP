# Chat 实施计划 v1

## 1. 文档定位

本文档定义 DeuteriumAPP 第一阶段 `chat` 模块的端到端实施计划，用于指导后续新窗口实现 App 与 Deuterium VIII 服务器主聊天的公共聊天互通、最近历史、实时收发、在线玩家侧边栏和服务器事件提示。

本文档遵循：

- `AGENTS.md`
- `CONTEXT.md`
- `docs/prd/account.md`
- `docs/prd/wallet-transfer.md`
- `docs/prd/chat.md`
- `docs/adr/0001-android-ui-stack.md`
- `docs/adr/0002-backend-plugin-bridge.md`
- `docs/adr/0003-account-identity-authority.md`
- `docs/adr/0004-backend-plugin-tech-stack.md`
- `docs/plans/foundation.md`
- `docs/plans/account.md`
- `docs/plans/wallet-transfer.md`

本文档不定义具体 API 路径、请求字段、响应字段、错误码、WebSocket 消息 schema、数据库表结构、插件事件监听代码或 Android 页面代码。

当前版本日期：2026-04-29。

## 2. 本次目标

Chat 的目标是让已登录并绑定服务器身份的玩家在 Android App 中参与 Deuterium VIII 服务器主聊天，并查看当前服务器在线状态。

本计划覆盖：

- 进入聊天页时加载最近 100 条公共聊天。
- 停留聊天页时实时接收公共聊天。
- 从 App 发送公共聊天到服务器主聊天。
- App 消息在服务器内显示为普通游戏内聊天，不额外标记 App 来源。
- 可选服务器事件提示开关。
- 在线人数展示。
- 在线玩家侧边栏。
- 在线玩家基础资料与转账入口。
- 历史加载失败、实时连接中断、发送失败、插件桥不可用等状态。

明确不做：

- 不做私聊、群聊、多频道或跨服聊天。
- 不做系统通知、后台推送、图片、附件、语音或表情包。
- 不做 @ 提醒、撤回、编辑、禁言或审核后台。
- 不做完整历史搜索、分页、导出或长期归档。
- 不展示 App 在线用户列表。
- 不让 App 伪造发送者身份。
- 不创建具体数据库 schema、实时协议 schema 或接口字段。

## 3. 涉及端与职责

Android 端：

- 使用 Compose 实现聊天页、消息列表、输入框、服务器事件设置和在线玩家侧边栏。
- 加载最近 100 条公共聊天。
- 维护与后端的实时连接。
- 校验消息非空、非纯空白且不超过 256 字符。
- 展示连接状态、发送失败、历史加载失败和在线状态不可用。
- 从在线玩家资料入口跳转或进入转账流程。

后端：

- 鉴权 App 用户和会话。
- 提供最近公共聊天历史。
- 维护 App 与后端之间的实时连接。
- 接收插件桥上报的服务器公共聊天和服务器事件。
- 将 App 消息转发给插件桥，并确认发送者来自当前登录账号绑定服务器解析 UUID。
- 提供在线人数和在线玩家列表。

Minecraft 插件：

- 监听服务器公共聊天事件。
- 按白名单监听可选服务器事件，例如死亡信息和服务器 `say` 信息。
- 通过插件桥上报聊天、事件和在线状态。
- 接收后端下发的 App 公共聊天转发请求。
- 以受控方式让服务器内显示为普通游戏内聊天。

数据库：

- 保存最近公共聊天和必要事件记录，用于进入聊天页时展示最近 100 条。
- 保存必要审计信息。
- 具体保留时长、表结构和字段名由后续实现确认。

文档：

- 若实时协议、历史保留周期或事件白名单需要长期决策，应在本计划待确认项中明确，必要时新增 ADR。

## 4. 实时通信选择

App 与后端之间第一版计划使用 WebSocket 实时连接。

选择理由：

- Chat v1 需要实时接收服务器公共聊天。
- 同一连接可以承载新消息、连接状态、在线人数变化和发送结果。
- 后端技术栈 Ktor 原生支持 WebSocket。
- 插件桥已经采用 WebSocket，后端可以保持统一的实时通信模型。

边界规则：

- WebSocket 只用于 App 与后端之间的受控实时通道。
- 后端仍然负责 App 鉴权和消息校验。
- App 不直接连接 Minecraft 插件。
- Android 前台聊天页使用 WebSocket 实时接收，同时每 15 秒静默拉取最近 100 条公共聊天并按消息 ID 去重补齐。
- Android 从后台回到前台聊天页时重新建立 WebSocket，并立即补齐最近 100 条公共聊天。
- 后台、锁屏或系统杀进程期间不保证实时接收；恢复前台后只补齐现有接口能提供的最近 100 条范围内消息。
- 具体 WebSocket 消息 schema 不在本文档定义，应由后续实现最小化确认。

如果后续发现移动网络稳定性或后台限制导致 WebSocket 体验不足，可以通过新的 ADR 讨论轮询、推送或混合方案。

## 5. 端到端数据流

最近历史数据流：

1. 玩家进入聊天页。
2. Android 请求最近 100 条公共聊天。
3. 后端校验会话并读取最近公共聊天记录。
4. 后端返回消息列表。
5. Android 展示公共聊天上下文。
6. Android 建立或恢复实时连接。

服务器到 App 实时消息数据流：

1. Minecraft 服务器内玩家发送公共聊天。
2. 插件监听聊天事件并获得服务器解析身份和当前游戏内 ID。
3. 插件通过插件桥上报后端。
4. 后端记录必要历史并广播给已连接的 App 用户。
5. Android 在聊天流中展示新消息。

App 到服务器发送数据流：

1. 玩家在 Android 输入消息。
2. Android 校验消息非空、非纯空白且不超过 256 字符。
3. Android 通过后端实时连接或发送入口提交消息。
4. 后端校验会话和账号绑定服务器解析 UUID。
5. 后端通过插件桥请求插件把消息转发到服务器主聊天。
6. 插件以受控方式在服务器内显示为普通游戏内聊天。
7. 服务器聊天事件回流或后端确认后，Android 聊天流展示该消息。

服务器事件提示数据流：

1. 插件监听白名单内服务器事件。
2. 插件通过插件桥上报事件。
3. 后端按事件类型记录或转发。
4. Android 仅在用户开启服务器事件提示时展示事件。
5. 事件提示独占一行、居中显示，不伪装成玩家发言。

在线状态数据流：

1. 玩家进入聊天页。
2. Android 请求或通过实时连接接收在线人数。
3. 玩家打开在线玩家侧边栏。
4. 后端通过插件桥获取或订阅服务器在线玩家列表。
5. Android 展示服务器当前在线玩家。
6. 玩家点击在线玩家后进入基础资料与转账入口。

## 6. 关键实现规则

聊天范围：

- Chat v1 只做全服公共聊天互通。
- 普通聊天不区分来源，不标记 App 或游戏内来源。
- App 转发到服务器的广播格式固定为 `§x§b§1§f§7§f§f%player% §7: §f%message%`。
- 第一版不做私聊、群聊、多频道或跨服聊天。

历史与实时：

- 进入聊天页展示最近 100 条公共聊天。
- 停留聊天页时实时接收新公共聊天。
- 完整历史搜索和长期归档不在第一版范围内。

发送权限：

- 只有已登录且绑定服务器解析 UUID 的 App 用户可以发送。
- App 不得允许用户选择发送者。
- App 不得提交自称可信的 UUID。
- 后端必须从当前会话确认发送者身份。

内容规则：

- 单条 App 消息最多 256 字符。
- 空消息不得发送。
- 只包含空白字符的消息不得发送。
- 敏感词、禁言和审核后台不在第一版范围内，默认沿用服务器既有规则和后续服务端能力。

服务器事件提示：

- 默认关闭。
- 玩家可在 Chat 设置中开启或关闭。
- 开启后事件提示出现在聊天流中。
- 事件提示独占一行、居中显示，并使用带透明度气泡覆盖的提示信息样式。
- 事件提示不得伪装成普通玩家发言。

发送失败策略：

- 插件桥不可用、连接中断或发送失败时，不离线排队。
- 不自动重发。
- Android 保留输入内容，允许玩家手动修改或重试。

## 7. 关键失败场景

历史消息加载失败：

- Android 提示聊天记录加载失败或稍后重试。
- 不把失败展示为没有历史消息。

实时连接不可用：

- Android 前台聊天页通过 WebSocket 自动重连和 15 秒最近消息补齐保持可见消息尽量最新。
- 非阻断的断开或静默补齐失败不反复打扰用户。
- 发送失败、登录失效等影响用户操作的错误仍需要明确提示。
- 不把断开伪装成当前没有新消息；超过最近 100 条窗口的断线缺口需要后续后端增量接口才能完整补回。

插件桥不可用：

- 服务器聊天事件、在线人数、在线列表和 App 消息转发可能失败。
- App 发送消息直接失败。
- 不离线排队消息。

发送者身份不可确认：

- 未登录、未绑定身份或账号身份异常时，后端阻止发送。
- Android 提示需要登录、完成身份绑定或稍后重试。

消息内容不合法：

- 空消息、纯空白消息或超过 256 字符时阻止发送。
- Android 展示明确提示。

发送失败：

- Android 提示发送失败。
- 不自动重发。
- 保留输入内容。

在线状态不可用：

- 在线人数或在线玩家列表展示不可用状态。
- 不展示错误在线人数。
- 不影响普通聊天历史展示，除非聊天连接也不可用。

## 8. 与账号和钱包的接口边界

Account 模块提供：

- 当前登录会话。
- 账号绑定服务器解析 UUID。
- 当前游戏内 ID 和基础资料。

Chat 模块不负责：

- 注册、登录、密码重设。
- 修改 QQ 号。
- 钱包余额读取。
- 执行转账。

Wallet Transfer 模块提供：

- 从在线玩家资料入口进入转账流程。
- 收款方最终确认、金额校验、二次确认和转账执行。

Chat 到 Wallet Transfer 的衔接规则：

- 在线玩家只作为收款方候选。
- 转账前仍必须遵守 `wallet-transfer` 的身份确认和二次确认。
- Chat 不得绕过钱包转账模块直接发起资产变更。

## 9. 验收标准

Chat 实施完成后必须满足：

- Chat v1 只做全服公共聊天互通。
- 进入聊天页时展示最近 100 条公共聊天。
- 停留聊天页时能实时接收新的公共聊天。
- 已登录且绑定服务器身份的 App 用户可以发送公共聊天。
- 未登录或未绑定身份用户不能发送。
- App 单条消息最多 256 字符。
- 空消息和纯空白消息不能发送。
- App 发送到服务器的消息显示为 `§x§b§1§f§7§f§f%player% §7: §f%message%` 对应的普通游戏内聊天样式。
- App 内普通聊天不区分来源。
- 服务器事件提示默认关闭。
- 玩家可以开启或关闭服务器事件提示。
- 事件提示独占一行、居中显示，并不会伪装成玩家发言。
- 发送失败时不自动重发、不进入离线队列，并保留输入内容。
- 聊天页顶栏显示服务器在线人数。
- 玩家目录优先展示服务器当前在线玩家，其后展示已注册 App 玩家及 App 在线状态。
- 点击在线玩家可以打开基础资料与转账入口。
- 点击玩家可以关心或取消关心；关心关系跟随账号同步。
- 被关心玩家发送普通公共聊天时，App 在后台触发安卓通知；服务器事件不触发关心通知。
- App WebSocket 从聊天页提升为登录态 App 级连接，退后台不主动断开，但不使用前台服务常驻通知。
- 第一版不做私聊、群聊、多频道、撤回、禁言或审核后台。
- App 不伪造发送者身份。

## 10. 验证计划

文档阶段验证：

- 对照 `docs/prd/chat.md` 检查公共聊天、最近 100 条、实时、事件提示、在线列表和错误状态是否覆盖。
- 对照 ADR 0003 检查 App 发送者必须来自账号绑定服务器解析 UUID。
- 对照 ADR 0002 检查服务器聊天能力都通过后端和插件桥。

后续实现阶段建议测试：

- 最近 100 条公共聊天加载成功。
- 历史加载失败时展示可理解状态。
- 服务器内玩家发言同步到 App。
- App 发言转发到服务器主聊天。
- 未登录、未绑定身份不能发送。
- 空消息、纯空白、超长消息被阻止。
- 插件桥不可用时发送失败且不排队。
- 实时连接断开时展示连接状态。
- 服务器事件提示默认关闭。
- 开启事件提示后事件按提示样式展示。
- 在线人数和在线玩家列表加载成功。
- 在线状态不可用时不展示错误人数。
- 在线玩家入口进入转账流程且不能绕过二次确认。
- `聊天 -> 我的 -> 聊天` 后，击杀播报与普通聊天气泡不发生错层覆盖。
- 关心玩家后有提示，再次点击取消关心也有提示。
- 后台收到被关心玩家普通聊天时显示安卓通知，前台不弹通知。
- 玩家目录排序为服务器在线玩家优先，之后按 App 在线状态展示已注册玩家。

## 11. 待确认假设

- App 与后端 WebSocket 的鉴权方式。
- 最近 100 条公共聊天的数据来源和保留时长。
- 服务器事件提示的事件类型白名单。
- 死亡信息、服务器 `say` 信息等事件的文案映射。
- 在线玩家基础资料字段。
- 从在线玩家进入转账流程的导航和预填规则。
- 聊天消息服务端审计范围和保留周期。
- 后台实时接收采用尽量保活，不承诺系统杀进程后持续接收。

## 12. v1.1 实施补充

本轮实现把 Chat 的实时连接提升到登录态 App 级别。Android 进入前台时上报前台状态，退后台时上报后台状态，但不主动断开 WebSocket。后端据此维护 App 在线状态：前台为在线，退后台后按最后活跃时间展示为 5 分钟内、30 分钟内、1 天内或 1 天以上。

新增玩家目录接口和账号同步关心接口。旧的在线玩家接口保持只返回服务器在线玩家，兼容旧 App。玩家目录显示当前登录用户，但详情中禁用关心和转账。

通知规则为：关心玩家普通聊天默认通知，钱包变动通知由钱包设置总开关控制；两类通知均只在 App 后台触发。安卓通知权限在用户首次关心玩家或开启钱包通知时请求。
