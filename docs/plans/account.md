# Account 实施计划 v1

## 1. 文档定位

本文档定义 DeuteriumAPP 第一阶段 `account` 模块的端到端实施计划，用于指导后续新窗口实现注册、登录、验证码、密码重设和退出登录。

本文档遵循：

- `AGENTS.md`
- `CONTEXT.md`
- `docs/prd/account.md`
- `docs/adr/0001-android-ui-stack.md`
- `docs/adr/0002-backend-plugin-bridge.md`
- `docs/adr/0003-account-identity-authority.md`
- `docs/adr/0004-backend-plugin-tech-stack.md`
- `docs/plans/foundation.md`

本文档不定义具体 API 路径、请求字段、响应字段、错误码、数据库表结构或 Android 页面代码。

当前版本日期：2026-04-29。

## 2. 本次目标

Account 的目标是让 Deuterium VIII 玩家用服务器身份安全创建和使用 DeuteriumAPP 账号，并为钱包转账和聊天提供可信登录态与服务器解析 UUID。

本计划覆盖：

- 请求注册验证码。
- 提交注册。
- 登录。
- 请求密码重设验证码。
- 提交新密码。
- 退出登录。
- 会话失效和登录失败限制。
- QQ 号作为必填资料和登录标识。

明确不做：

- 不引入短信验证码、邮箱验证、OAuth 或第三方登录。
- 不设计管理员人工迁移工具。
- 不实现 QQ 号修改或隐藏开关。
- 不让 App 直连数据库。
- 不把游戏内 ID 当作权威身份。
- 不创建具体数据库 schema 或接口字段。

## 3. 涉及端与职责

Android 端：

- 使用 Compose 页面承载注册、登录、验证码输入、密码重设和退出登录流程。
- 在本地执行基础输入校验，例如空值、密码长度、验证码格式。
- 调用后端 API，不接触数据库、插件桥密钥或服务器凭据。
- 对玩家展示清晰错误状态，例如玩家不在线、验证码错误、登录限制中。

后端：

- 提供账号相关业务能力。
- 负责密码哈希、验证码生成与校验、登录失败限制、会话签发与失效。
- 通过插件桥解析游戏内 ID 对应的服务器解析 UUID。
- 通过插件桥请求服务器向在线玩家发送私聊验证码。
- 维护 QQ 号唯一性和 UUID 唯一性。

Minecraft 插件：

- 接收后端受控请求。
- 查询玩家是否在线。
- 解析游戏内 ID 对应服务器解析 UUID。
- 向在线玩家发送验证码私聊消息。
- 不接收 App 直连请求，不处理公网账号 API。

数据库：

- 保存账号绑定关系、密码凭据、QQ 号、当前游戏内 ID、验证码状态、登录失败限制和会话状态。
- 具体表结构和字段名由后续实现确认。
- 不保存明文密码。

文档：

- 若实现中发现会话 token 策略、密码哈希算法或登录限制维度需要长期决策，应补充安全 ADR 或在本计划中记录待确认项。

## 4. 端到端数据流

注册验证码数据流：

1. 玩家在 Android 输入游戏内 ID、QQ 号和密码，并请求验证码。
2. 后端校验基础输入和冷却限制。
3. 后端通过插件桥请求插件确认玩家在线并解析服务器 UUID。
4. 插件向该在线玩家发送 6 位数字验证码私聊。
5. 后端保存验证码用途、过期时间、尝试次数和绑定上下文。
6. Android 展示验证码已发送。

提交注册数据流：

1. 玩家在 Android 输入验证码并提交注册。
2. 后端校验验证码未过期、未超过 5 次尝试、用途匹配且玩家身份上下文一致。
3. 后端确认 UUID 未注册、QQ 号未绑定其他账号、密码长度符合 8-64。
4. 后端保存账号、UUID 绑定、当前游戏内 ID、QQ 号和密码哈希。
5. 后端签发当前设备会话。
6. Android 进入登录态。

登录数据流：

1. 玩家在单一账号输入框输入玩家 ID 或 QQ 号，并输入密码。
2. 后端识别登录标识并查找账号。
3. 后端校验密码和登录失败限制。
4. 登录成功时，后端签发当前设备会话并清理相关失败状态。
5. Android 进入登录态。

密码重设数据流：

1. 玩家输入玩家 ID 或 QQ 号，请求密码重设验证码。
2. 后端找到已绑定账号和服务器 UUID。
3. 后端通过插件桥确认该绑定玩家在线，并请求插件发送验证码。
4. 玩家提交验证码和新密码。
5. 后端校验验证码和密码长度。
6. 后端更新密码哈希，并使该账号既有会话全部失效。
7. Android 引导玩家使用新密码重新登录。

退出登录数据流：

1. 玩家在 Android 触发退出登录。
2. 后端或 Android 清除当前设备会话状态。
3. Android 返回未登录状态。

## 5. 关键实现规则

身份规则：

- 服务器解析 UUID 是账号绑定权威。
- 游戏内 ID 是输入、展示和当前登录别名。
- QQ 号是必填资料和登录标识，不是 Minecraft 身份权威。
- App 不得提交可信 UUID。

验证码规则：

- 验证码为 6 位数字。
- 有效期 10 分钟。
- 重发冷却 60 秒。
- 单个验证码最多尝试 5 次。
- 新验证码生成后，旧验证码立即失效。
- 验证码只通过服务器私聊发送，不在 App 响应中返回明文。
- 注册验证码和密码重设验证码不能跨用途复用。

密码与登录规则：

- 密码长度 8-64。
- 不保存明文密码。
- 连续 5 次登录失败后，临时限制 15 分钟。
- 登录失败提示不暴露过多账号枚举信息。
- 密码重设成功后，该账号既有会话全部失效。

QQ 号规则：

- 注册时必填。
- 全局唯一。
- 可作为登录标识。
- 第一版资料页对已登录 App 用户可见。
- QQ 修改和隐藏不在本计划范围内。

## 6. 关键失败场景

玩家不在线：

- 注册或密码重设验证码发送失败。
- 后端不生成可用验证码。
- Android 提示玩家先进入 Deuterium VIII 服务器。

游戏内 ID 不存在或身份解析失败：

- 阻止验证码发送或注册绑定。
- Android 提示检查游戏内 ID 或稍后重试。

UUID 已注册：

- 阻止重复注册。
- Android 引导玩家登录或走密码重设。

QQ 号重复：

- 阻止注册。
- Android 提示 QQ 号已被使用。

验证码错误、过期或次数耗尽：

- 错误时记录尝试次数。
- 过期或次数耗尽时要求重新获取验证码。
- 不允许继续使用失效验证码。

账号或密码错误：

- 登录失败。
- 提示账号或密码错误。
- 达到失败上限后进入 15 分钟临时限制。

插件桥不可用：

- 验证码发送和身份解析失败。
- Android 提示服务器连接暂不可用。

数据库不可用：

- 后端账号流程失败。
- 不返回伪成功状态。

## 7. 与钱包和聊天的接口边界

钱包转账模块依赖：

- 当前登录账号。
- 账号绑定的服务器解析 UUID。
- 当前游戏内 ID 作为展示名称。

聊天模块依赖：

- 当前登录账号。
- 账号绑定的服务器解析 UUID。
- 当前游戏内 ID 作为展示名称。
- 会话有效性判断。

Account 模块不负责：

- 获取信用点余额。
- 执行信用点转账。
- 同步聊天消息。
- 管理在线玩家列表。

## 8. 验收标准

Account 实施完成后必须满足：

- 在线玩家可以通过游戏内 ID、QQ 号、密码和服务器私聊验证码完成注册。
- 注册成功后账号绑定服务器解析 UUID。
- 同一 UUID 不能重复注册多个账号。
- 同一 QQ 号不能绑定多个账号。
- 玩家可以用玩家 ID + 密码登录。
- 玩家可以用 QQ 号 + 密码登录。
- 登录页使用单一账号输入框。
- 玩家不在线时不能获取注册或改密验证码。
- 验证码符合 6 位数字、10 分钟有效期、60 秒冷却、5 次尝试上限。
- 新验证码生成后旧验证码失效。
- 密码长度限制为 8-64。
- 连续 5 次登录失败后限制 15 分钟。
- 密码重设成功后既有会话全部失效。
- App 不直连数据库，不直接连接插件桥。
- 文档和实现不引入短信、邮箱、OAuth 或第三方登录。

## 9. 验证计划

文档阶段验证：

- 对照 `docs/prd/account.md` 检查所有产品级概念操作都被覆盖。
- 对照 ADR 0003 检查没有把游戏内 ID 当作权威身份。
- 对照 ADR 0002 检查验证码发送必须经过后端和插件桥。

后续实现阶段建议测试：

- 注册成功。
- 玩家不在线无法发送验证码。
- UUID 已注册阻止重复注册。
- QQ 重复阻止注册。
- 验证码错误、过期、次数耗尽。
- 玩家 ID 登录成功。
- QQ 登录成功。
- 登录失败限制。
- 密码重设后旧密码和旧会话失效。
- 插件桥不可用时账号流程失败并展示可理解提示。

## 10. 待确认假设

- 会话 token 的类型、有效期、刷新机制和存储方式。
- 登录失败限制按账号、设备、IP 还是组合维度执行。
- 密码哈希算法和参数。
- 游戏内 ID 大小写和当前显示名更新策略。
- 历史游戏内 ID 是否允许登录。
- QQ 号展示范围是否需要在 v1 之外继续收紧。
- 插件桥发送验证码的具体文案。
