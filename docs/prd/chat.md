# 聊天互通 PRD v1

## 1. 文档定位

本文档定义 DeuteriumAPP 第一版 `chat` 模块的产品需求，覆盖 App 与 Deuterium VIII 服务器主聊天的公共聊天互通、最近历史消息、实时收发、在线人数、在线玩家侧边栏、基础资料与转账入口、可选服务器事件提示和关键错误状态。

本文档遵循根目录 `AGENTS.md`、`CONTEXT.md`、`docs/prd/account.md`、`docs/prd/wallet-transfer.md` 以及已有 ADR 的约束：先文档后代码、服务 Deuterium VIII、后端通过插件桥与服务器交互、插件桥不可用时消息不离线排队、玩家身份以服务器解析 UUID 为权威。

本文档不定义数据库结构、具体 API 路径、请求字段、响应字段、错误码、App 与后端实时通信协议、WebSocket 消息 schema、插件事件监听细节或 Android UI 实现。这些内容应在后续 ADR、实现计划或接口文档中确认。

当前版本日期：2026-04-29。

## 2. 目标

聊天互通的目标是让 Deuterium VIII 玩家在 Android App 中参与服务器主聊天，即使不在 Minecraft 客户端内，也能看到公共聊天、发送公共聊天，并了解当前服务器在线情况。

第一版需要做到：

- 玩家可以在 App 中看到服务器最近公共聊天上下文。
- 玩家可以在 App 聊天页内实时接收新的公共聊天消息。
- 已登录且绑定服务器身份的玩家可以从 App 发送公共聊天消息到服务器。
- App 发送到服务器的消息在服务器内显示为普通游戏内聊天。
- App 聊天室默认只展示玩家公共聊天。
- 玩家可以在聊天室设置中开启服务器事件提示，查看死亡信息、服务器 `say` 信息等非玩家公共聊天事件。
- 玩家可以看到服务器在线人数，并通过侧边栏查看在线玩家列表。
- 玩家可以从在线玩家列表进入基础资料与转账入口。
- 系统能明确反馈插件桥不可用、发送失败、消息过长、身份不可确认等状态。

## 3. 用户与场景

### 3.1 主要用户

主要用户是已经注册并登录 DeuteriumAPP 的 Deuterium VIII 玩家。玩家已经通过账号体系绑定服务器解析 UUID，并希望在离开 Minecraft 客户端后继续参与服务器公共聊天。

### 3.2 核心场景

- 玩家不在电脑前，想看服务器里大家正在聊什么。
- 玩家打开聊天页，希望先看到最近一小段聊天上下文。
- 玩家想在 App 中发一句话，让服务器里的玩家像看到普通游戏内聊天一样看到它。
- 玩家想知道当前服务器有多少人在线。
- 玩家想打开在线玩家侧边栏，看当前有哪些玩家。
- 玩家想点击在线玩家查看基础资料，并快速进入转账流程。
- 玩家希望默认聊天列表保持干净，不被死亡信息或服务器 `say` 信息打断。
- 玩家在需要时，可以打开聊天室设置，开启服务器事件提示。

## 4. 核心决策

### 4.1 聊天范围

Chat v1 只做全服公共聊天互通。

公共聊天指 Deuterium VIII 服务器主聊天中的玩家公共发言，以及 App 用户发送到服务器主聊天的公共发言。

第一版不做私聊、群聊、多频道或跨服聊天。

### 4.2 最近历史与实时消息

玩家进入聊天页时，App 展示最近 100 条公共聊天消息。

玩家停留在聊天页时，App 继续实时接收新的公共聊天消息。

完整历史搜索、分页、导出或长期聊天归档不在第一版范围内。

### 4.3 消息发送权限

只有已登录且绑定服务器解析 UUID 的 App 用户可以发送公共聊天消息。

未登录用户、未完成身份绑定的用户或身份不可确认的用户不得发送聊天消息。

App 不得允许用户手动选择发送者，也不得允许用户指定任意游戏内 ID 或 UUID 来伪造身份。

### 4.4 消息显示规则

App 发送到服务器的消息，在服务器内显示为普通游戏内聊天，不额外标记 App 来源。

App 内普通聊天消息不区分来源，只展示统一公共聊天室体验。来自服务器内玩家和来自 App 玩家发送的普通公共聊天，在 App 中不做来源标记。

即使显示上不区分来源，发送者身份仍必须由服务器解析 UUID 和账号绑定关系保证。

### 4.5 消息内容规则

- 单条 App 发送消息最多 256 字符。
- 空消息不得发送。
- 只包含空白字符的消息不得发送。
- 超过长度限制的消息不得发送。

敏感词、禁言、撤回、编辑和消息审核后台不在 Chat v1 范围内。第一版内容治理默认沿用服务器既有规则和后续服务端能力。

### 4.6 服务器事件提示

Chat v1 支持可选服务器事件提示。

服务器事件提示用于展示死亡信息、服务器 `say` 信息等非玩家公共聊天事件。它们不是普通玩家聊天消息，不参与普通聊天消息样式。

服务器事件提示规则如下：

- 默认关闭。
- 玩家可以在 Chat 聊天室设置中开启或关闭。
- 开启后，事件提示出现在聊天流中。
- 事件提示独占一行。
- 事件提示居中显示。
- 事件提示使用带透明度气泡覆盖的提示信息样式。
- 事件提示不得伪装成玩家发言。

具体事件类型白名单、事件文案映射、服务器事件来源和插件消息格式不在 PRD v1 中定义，应在后续实现计划中确认。

### 4.7 发送失败策略

当插件桥不可用、连接中断或发送失败时，App 消息不进入离线队列，也不自动重发。

发送失败时，App 应提示发送失败，并保留输入内容，允许玩家手动修改或重试。

### 4.8 在线状态与侧边栏

聊天页顶栏显示轻量在线人数。

玩家可以从聊天页打开在线玩家侧边栏，查看 Deuterium VIII 服务器当前在线玩家列表。

在线玩家列表展示服务器当前在线玩家，不是 App 在线用户列表。

玩家点击在线列表中的玩家后，可以打开基础资料与转账入口。

基础资料字段遵守账号 PRD 既有隐私边界。Chat PRD 不扩展完整资料页体系，也不定义资料页字段细节。

转账入口应与 `wallet-transfer` 模块衔接。具体跳转、预填和 UI 交互由后续实现计划确认。

## 5. 功能需求

### 5.1 查看最近公共聊天

#### 用户目标

玩家进入聊天页后，可以快速了解服务器最近聊天上下文。

#### 前置条件

- 玩家已登录 App。
- 后端可提供最近公共聊天消息。

#### 基本流程

1. 玩家进入聊天页。
2. App 请求最近公共聊天消息。
3. App 展示最近 100 条公共聊天消息。
4. App 进入实时接收状态。

#### 验收标准

- 玩家进入聊天页时可以看到最近 100 条公共聊天消息。
- 普通聊天消息展示玩家名、消息内容和时间等用户可理解信息。
- 最近历史只要求公共聊天消息，不默认包含死亡信息、服务器 `say` 信息或其他服务器事件。
- 历史加载失败时，玩家能看到可理解的失败状态。

### 5.2 实时接收公共聊天

#### 用户目标

玩家停留在聊天页时，可以看到服务器新发出的公共聊天消息。

#### 基本流程

1. 玩家停留在聊天页。
2. 服务器内玩家或 App 玩家发送公共聊天。
3. 系统将公共聊天同步到 App。
4. App 在聊天流中展示新消息。

#### 验收标准

- App 能接收新的公共聊天消息。
- App 普通聊天不区分来源，只展示统一聊天室体验。
- 消息发送者身份以服务器侧身份为准。
- 连接中断时，App 应展示连接不可用或正在恢复的状态。

### 5.3 从 App 发送公共聊天

#### 用户目标

玩家可以在 App 中发送公共聊天消息，让服务器内玩家看到。

#### 前置条件

- 玩家已登录 App。
- 玩家账号已绑定服务器解析 UUID。
- 插件桥可用。

#### 基本流程

1. 玩家在聊天输入框输入消息。
2. App 校验消息非空且不超过 256 字符。
3. 玩家发送消息。
4. 系统确认当前登录玩家身份。
5. 系统把消息转发到 Deuterium VIII 服务器主聊天。
6. 服务器内显示为普通游戏内聊天。
7. App 聊天流中也展示该消息。

#### 验收标准

- 未登录用户不能发送消息。
- 未绑定服务器身份的用户不能发送消息。
- 空消息不能发送。
- 超过 256 字符的消息不能发送。
- App 不能伪造其他玩家身份。
- App 消息进入服务器后不额外标记 App 来源。
- 发送失败时不自动重发，也不进入离线队列。

### 5.4 切换服务器事件提示

#### 用户目标

玩家可以按自己的偏好决定是否在聊天室中看到死亡信息、服务器 `say` 信息等事件提示。

#### 基本流程

1. 玩家打开 Chat 聊天室设置。
2. 玩家开启或关闭服务器事件提示。
3. 设置关闭时，聊天流只展示普通公共聊天消息。
4. 设置开启时，聊天流中可展示服务器事件提示。

#### 验收标准

- 服务器事件提示默认关闭。
- 玩家可以在 Chat 设置中开启或关闭。
- 开启后，事件提示独占一行、居中显示。
- 事件提示使用带透明度气泡覆盖的提示信息样式。
- 事件提示不会展示成玩家普通聊天消息。
- 关闭后，事件提示不应继续混入普通聊天流。

### 5.5 查看在线人数和在线玩家列表

#### 用户目标

玩家可以在聊天页知道当前服务器在线情况，并查看当前在线玩家。

#### 基本流程

1. 玩家进入聊天页。
2. 顶栏展示当前在线人数。
3. 玩家打开在线玩家侧边栏。
4. App 展示当前在线玩家列表。

#### 验收标准

- 聊天页顶栏显示在线人数。
- 在线人数表示 Deuterium VIII 服务器当前在线玩家数。
- 在线玩家侧边栏展示当前服务器在线玩家列表。
- 在线玩家列表不是 App 在线用户列表。
- 在线状态不可用时，玩家能看到可理解的失败或不可用状态。

### 5.6 查看在线玩家基础资料并进入转账

#### 用户目标

玩家可以从在线列表中快速查看某个玩家的基础信息，并进入转账流程。

#### 基本流程

1. 玩家打开在线玩家侧边栏。
2. 玩家点击某个在线玩家。
3. App 展示该玩家基础资料和转账入口。
4. 玩家点击转账入口。
5. App 进入或跳转到 `wallet-transfer` 转账流程。

#### 验收标准

- 点击在线玩家可以打开基础资料与转账入口。
- 基础资料不突破账号 PRD 的隐私边界。
- 转账入口与 `wallet-transfer` 模块衔接。
- 第一版不提供私聊入口。

## 6. 关键错误状态

### 6.1 历史消息加载失败

触发场景：

- 网络不可用。
- 后端不可用。
- 最近历史无法读取。

产品要求：

- 提示聊天记录加载失败或稍后重试。
- 不阻止用户继续尝试实时连接或发送，除非实时连接也不可用。

### 6.2 实时连接不可用

触发场景：

- App 与后端实时连接不可用。
- 后端无法继续向 App 推送或同步消息。

产品要求：

- 展示连接不可用、正在重连或稍后再试等可理解状态。
- 不把连接断开伪装成“当前没有新消息”。

### 6.3 插件桥不可用

触发场景：

- 后端无法通过插件桥接收服务器聊天事件。
- 后端无法通过插件桥把 App 消息转发到服务器。
- 后端无法通过插件桥获取在线人数或在线玩家列表。

产品要求：

- 发送消息失败。
- 在线状态可能不可用。
- 提示服务器连接暂不可用，请稍后再试。
- 不要求玩家理解插件桥细节。
- 不离线排队消息。

### 6.4 发送者身份不可确认

触发场景：

- 玩家未登录。
- 玩家未绑定服务器身份。
- 当前账号绑定身份异常。
- 后端无法确认当前登录账号对应服务器解析 UUID。

产品要求：

- 阻止发送消息。
- 提示需要登录、完成身份绑定或稍后重试。
- 不允许 App 自行指定发送者。

### 6.5 消息内容不合法

触发场景：

- 消息为空。
- 消息只包含空白字符。
- 消息超过 256 字符。

产品要求：

- 阻止发送。
- 明确提示消息不能为空或超出长度限制。

### 6.6 发送失败

触发场景：

- 网络中断。
- 后端拒绝发送。
- 插件桥不可用。
- 服务器侧发送失败。

产品要求：

- 提示发送失败。
- 不自动重发。
- 不进入离线队列。
- 保留输入内容，允许玩家手动修改或重试。

### 6.7 在线状态不可用

触发场景：

- 后端无法获取在线人数。
- 后端无法获取在线玩家列表。
- 插件桥不可用。

产品要求：

- 在线人数或列表展示为不可用状态。
- 不展示错误的在线人数。
- 不影响普通聊天历史展示，除非聊天连接也不可用。

## 7. 与账号、钱包和插件桥的关系

### 7.1 账号身份

Chat v1 发送消息必须依赖当前登录账号绑定的服务器解析 UUID。

App 不得允许用户手动填写发送者游戏内 ID，也不得允许用户伪造 UUID。

### 7.2 钱包转账

在线玩家侧边栏中的转账入口应进入 `wallet-transfer` 模块。

从聊天侧边栏发起转账时，收款方来自被点击的在线玩家，但真正执行转账前仍必须遵守 `wallet-transfer` PRD 的收款方确认和二次确认规则。

### 7.3 插件桥

服务器公共聊天消息、服务器事件提示、在线人数、在线玩家列表和 App 消息转发都依赖后端与 Minecraft 插件桥。

插件桥不可用时，不离线排队聊天消息，也不自动补发。

## 8. 非目标

聊天互通 PRD v1 不包含以下内容：

- 私聊。
- 群聊。
- 多频道。
- 跨服聊天。
- 敏感词过滤。
- 禁言。
- 撤回。
- 编辑。
- 消息审核后台。
- 手机系统通知。
- 后台推送。
- 图片、附件、语音、表情包。
- @ 提醒。
- 完整历史搜索、分页、导出或长期聊天归档。
- App 在线用户列表。
- 完整资料页体系。
- 数据库表结构。
- 具体 API 路径、请求字段、响应字段或错误码。
- App 与后端实时通信协议。
- WebSocket 消息 schema。
- 插件事件监听细节。
- Android UI 技术实现。

## 9. 产品级概念操作

本文档只定义以下产品级概念操作：

- 查看最近公共聊天。
- 实时接收公共聊天。
- 发送公共聊天。
- 开启或关闭服务器事件提示。
- 查看在线人数。
- 查看在线玩家列表。
- 查看在线玩家基础资料。
- 从在线玩家进入转账流程。

这些操作的接口路径、参数、错误码、存储模型、实时通信协议、插件桥消息格式和 Android 实现由后续 ADR 或实现计划定义。

## 10. 验收清单

- Chat v1 是否只做全服公共聊天互通。

- App 是否能实时接收新的公共聊天消息。
